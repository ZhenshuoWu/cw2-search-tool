# CW2 Search Tool

This project is a Python command-line search tool for
`https://quotes.toscrape.com/`. It crawls the quote listing pages, builds a
case-insensitive inverted index, saves that index to disk, reloads it later,
and lets users inspect or search the index from an interactive shell.

## Features

- Polite crawler for `quotes.toscrape.com` pagination, with a minimum 6 second
  delay between successive real requests.
- Inverted index storing each word's page frequency and token positions.
- JSON persistence through `data/index.json`, plus a crawl audit report in
  `data/crawl_report.json`.
- Required shell commands: `build`, `load`, `print <word>`, and
  `find <query terms>`.
- Ranked search using BM25 scoring, with highlighted snippets and token
  position evidence.
- Exact phrase search using quotes, for example `find "good friends"`.
- Spelling suggestions for missing query terms, for example
  `find indiference`.
- Optional trace visualizer for explaining extraction, tokenization, indexing,
  and query intersection in the video demonstration.
- Reproducible benchmark script for indexing and ranked search performance.

## Setup

Create and activate a virtual environment, then install the dependencies:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Running the Shell

Start the interactive shell:

```bash
python -m src.main
```

Example session:

```text
> build
Index ready: 10 pages, 625 unique words.
Saved index to data/index.json.
Crawl report saved to data/crawl_report.json: 10 requests, 0 errors.

> load
Index ready: 10 pages, 625 unique words.
Loaded index from data/index.json.

> print nonsense
Index entry for 'nonsense':
- https://quotes.toscrape.com/page/8/: frequency=1, positions=[...]

> find indifference
Pages matching indifference (ranked by BM25):
1. https://quotes.toscrape.com/page/1/
   score=...
   snippet: ... [indifference] ...

> find good friends
Pages matching good friends (ranked by BM25):
1. https://quotes.toscrape.com/page/1/
   score=...
   snippet: ... [good] ... [friends] ...

> find "good friends"
```

`build` performs real network requests and therefore takes about one minute for
the full site because the crawler respects the 6 second politeness window.

## Trace Visualizer

Run the demo visualizer:

```bash
python -m src.trace_web
```

Then open `http://127.0.0.1:8765/`.

After running `build`, the visualizer can also render the saved index:

```bash
python -m src.trace_web --index data/index.json
```

This is useful for the video walkthrough because it shows the pipeline from
HTML extraction to tokens, index updates, and query set intersection.

## Project Structure

```text

  crawler.py      # polite crawling and crawl audit data
  indexer.py      # quote extraction, tokenization, inverted index building
  search.py       # BM25 ranking, snippets, phrase search, suggestions
  storage.py      # JSON save/load for the index and crawl report
  main.py         # interactive command-line shell
  trace_web.py    # optional web visualizer for explanation
scripts/
  benchmark_search.py
  demo_crawl.py
tests/
  test_crawler.py
  test_indexer.py
  test_search.py
  test_storage.py
  test_trace_web.py
data/
  index.json          # generated by build, ignored by git
  crawl_report.json   # generated by build, ignored by git
requirements.txt
README.md
```

## Testing

Run the full test suite:

```bash
pytest
```

Run tests with coverage:

```bash
pytest --cov=src
```

The tests use fake sessions and injected pages, so they do not depend on the
live website and do not wait for the real 6 second politeness delay.

## Benchmarking

Run a deterministic benchmark without network access:

```bash
python scripts/benchmark_search.py --pages 500 --repeat 5
```

For a larger stress test:

```bash
python scripts/benchmark_search.py --pages 5000 --repeat 3
```

The benchmark reports median index-build time and median ranked-query time over
synthetic quote-like pages. This gives evidence for the complexity discussion
without depending on the live website or network speed.

## Design Notes

The index uses a nested dictionary:

```text
word -> page URL -> frequency and positions
```

This structure makes `print <word>` a direct dictionary lookup and makes
multi-word search efficient by intersecting the page sets for each query term.
Positions are stored so the tool can support exact phrase queries and explain
matches in the output.

BM25 ranking is applied after the required page-set intersection. This keeps
the required behavior simple and correct while adding a defensible ranking
model for the higher-grade search features.

More detailed complexity analysis, benchmark interpretation, and tradeoffs are
documented in `docs/engineering_notes.md`.
